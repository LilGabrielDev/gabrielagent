# WhatsApp Channel

## Overview

The WhatsApp channel connects Gabriel to customer messages on WhatsApp. Gabriel supports three connection methods from the **Channels** page:

| Method | Best for | Runtime |
|--------|----------|---------|
| **WhatsApp Web (QR)** | Quick setup, personal or business accounts | External `whatsapp-service` |
| **Pairing Code** | Servers without camera; Linked Devices flow | External `whatsapp-service` |
| **Business API** | Production scale via Meta | Credentials stored in Gabriel; Meta webhooks |

## Architecture (Split Deployment)

Gabriel's dashboard and API run on **Vercel** (or Docker/Kubernetes). WhatsApp sessions run on a **separate long-lived service** because Vercel cannot host Baileys/WebSocket sessions reliably.

```
Dashboard (Vercel)  --WHATSAPP_SERVICE_URL-->  whatsapp-service (Render / Railway / Katabump / VPS)
                                                      |
                                                      v
                                               WhatsApp (Baileys)
```

### Environment variables

**Gabriel (main app)**

| Variable | Required | Description |
|----------|----------|-------------|
| `WHATSAPP_SERVICE_URL` | Yes (for Web/Pair) | Public URL of your deployed whatsapp-service |
| `WHATSAPP_SERVICE_API_KEY` | Recommended | Must match `API_KEY` on the service |

**whatsapp-service**

| Variable | Required | Description |
|----------|----------|-------------|
| `PORT` | Auto on PaaS | Listen port |
| `SESSION_PATH` | Yes in production | Persistent volume path (e.g. `/app/sessions`) |
| `API_KEY` | Recommended | Locks down the service |
| `ALLOWED_ORIGINS` | Optional | Comma-separated CORS origins (include your Vercel URL) |

See `whatsapp-service/README.md` for Render, Railway, and Katabump deploy steps.

## Method 1: WhatsApp Web (QR Code)

1. Deploy `whatsapp-service` and set `WHATSAPP_SERVICE_URL` on Gabriel.
2. Open **Channels** → WhatsApp → **WhatsApp Web**.
3. Click **Connect**. A QR code appears (generated by Baileys on the service).
4. On your phone: **Settings → Linked Devices → Link a Device** and scan the QR.
5. Status progresses: `connecting` → `qr_ready` → `connected`.

Sessions persist in `SESSION_PATH` on the service host. Remount the same volume after restarts to avoid re-scanning.

---

## Method 2: Pairing Code

1. Deploy `whatsapp-service` and configure `WHATSAPP_SERVICE_URL`.
2. Open **Channels** → WhatsApp → **Pairing Code**.
3. Enter your WhatsApp phone number (country code, digits only, e.g. `6281376552730`).
4. Click **Get Pairing Code**. An 8-character code appears.
5. On your phone: **Linked Devices → Link a Device → Link with phone number instead** and enter the code.
6. Status: `pairing_ready` → `connected`.

---

## Method 3: WhatsApp Business API

1. Open **Channels** → WhatsApp → **API**.
2. Enter your Meta Business API key and registered phone number.
3. Click **Save API Credentials**.

This stores credentials in the database. Full inbound/outbound messaging via Meta's Cloud API requires configuring Meta webhooks to your Gabriel deployment (see [Webhooks](Webhooks)).

## API Endpoints

### Gabriel (proxied to service)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/channels/whatsapp` | Current QR/pair/connection status |
| POST | `/api/channels/whatsapp` | `connect`, `disconnect`, or `sync` |

### whatsapp-service (direct)

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/whatsapp/qr` | Start QR session |
| POST | `/api/whatsapp/pair` | Request pairing code |
| GET | `/api/whatsapp/status/:sessionId` | Session status |
| DELETE | `/api/whatsapp/logout/:sessionId` | Logout and clear session |
| POST | `/api/whatsapp/reconnect/:sessionId` | Restore saved session |
| GET | `/health` | Health check |

## Disconnecting

Click **Disconnect** on the Channels card. Gabriel calls the service logout endpoint and updates the channel record to `disconnected`.

## Troubleshooting

### "WHATSAPP_SERVICE_URL is not configured"

Set the variable on Vercel (or in `.env` locally) to your deployed service URL.

### QR or pairing code never appears

- Confirm the service health endpoint: `GET https://your-service/health`
- Check `ALLOWED_ORIGINS` includes your dashboard URL
- Verify `WHATSAPP_SERVICE_API_KEY` matches on both sides
- Inspect service logs for Baileys connection errors

### Connection drops after deploy

Ensure `SESSION_PATH` is on a **persistent volume**, not ephemeral disk.

### Authentication errors on Channels page

Ensure you are logged in (`gabriel-token` cookie). API routes require authentication.
